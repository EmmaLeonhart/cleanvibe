cleanvibe, a state initializer

A tiny, zero-dependency Python CLI that removes the friction between “I want to build something” and “Claude is working inside a well-structured environment.” It scaffolds the project, injects an opinionated CLAUDE.md behavior contract, and launches Claude Code.

pip install cleanvibe GitHub

01 — What it is

cleanvibe is not a coding tool — it is a state initializer. The real value lives in the CLAUDE.md it injects: a behavior contract that enforces documentation discipline, meaningful commits, and iterative file-based thinking, plus a queue.md pre-seeded with a first-session bootstrap sequence. Every project starts disciplined instead of as a blank-repo scramble. Zero runtime dependencies; works on Windows, Linux, and macOS; Python 3.9+.

02 — The subcommands

🌱 cleanvibe new

Scaffold a fresh project: CLAUDE.md, README.md, queue.md (pre-seeded bootstrap), .gitignore, an initial git commit, then launch Claude Code inside it.

🔗 cleanvibe clone

Onboard an existing repo. Works on a dedicated cleanvibe-onboarding branch — the default branch is untouched. Prepends scaffold if files exist; never overwrites.

♻ cleanvibe convert

In-place onboarding for a project you are already in: injects only the scaffold files that are missing. Non-destructive by contract.

📚 cleanvibe replicate

Point it at an arXiv or alphaxiv URL and it scaffolds a standalone paper-replication project: SKILL.md plan, the paper as a submodule target, and CI that builds a findings site, a PDF report, and a ZIP package.

03 — Stable since v1.0.0

Semantic versioning from here on. new, clone, convert, and replicate are stable within the 1.x line; the guaranteed set of injected files is fixed; clone/convert never overwrite and replicate never errors on a name collision; zero runtime dependencies is a hard guarantee. Preview anything with --dry-run.
